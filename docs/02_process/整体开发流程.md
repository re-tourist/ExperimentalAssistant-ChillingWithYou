# 整体开发流程（最终冻结版）

> 本文档用于指导本课程大作业的**完整开发流程**，作为项目的总纲文件。
>  流程强调：**先设计、再开发、过程可追踪、结果可验收**，并适配 AI 辅助开发。

------

## 一、项目概述与总体目标

### 1.1 项目主题

**项目 / 实验数据管理与可视化平台（Project & Experiment Dashboard）**

系统用于管理项目/实验过程中的数据记录（Run、Metrics、Tags 等），并通过数据可视化 Dashboard 展示实验指标的趋势、分布与对比结果，整体风格参考深度学习实验管理与分析场景。

### 1.2 系统核心结构

- **主线**：数据可视化 Dashboard（分析、统计、趋势、TopN）
- **辅线**：资源 / 内容管理（Project、Run、Metrics、Tags）
- **目标**：形成一个数据来源清晰、分析逻辑明确、界面完整的 Web 系统

## 二、技术选型（Frozen，不再变更）

> 本节为**冻结决策**，后续开发与文档以此为唯一技术依据。

### 2.1 前端技术栈

- 框架：**Vue 3（Composition API）**
- 构建工具：**Vite**
- UI 组件库：**Element Plus**
- 路由管理：**Vue Router**
- 状态管理：**Pinia**
- HTTP 请求：**Axios（统一拦截与响应封装）**
- 图表可视化：**ECharts**

### 2.2 后端技术栈

- 框架：**Spring Boot 3（Java 17）**
- 数据访问层：**MyBatis-Plus**
- 数据库：**MySQL**
- 校验：**Spring Validation**
- 统计策略：
  - **CRUD 使用 MyBatis-Plus**
  - **Dashboard 聚合查询使用自定义 Mapper SQL（XML / 注解）**

### 2.3 技术选型说明（写给老师）

- Vue + Element Plus 适合构建后台管理与数据看板类系统，开发效率高、界面成熟；
- Spring Boot + MyBatis-Plus 适合快速实现分页、事务与统计查询；
- 通过分离 CRUD 与统计聚合逻辑，提高系统结构清晰度与可维护性。

### 3.3 Commit 规范

## 四、Issue 驱动的任务管理（最小实践）

### 4.1 Issue 的作用

Issue 用于拆分任务、记录过程与分工，是项目过程可追踪性的体现。

### 4.2 使用方式（最小但有效）

- 每个开发里程碑对应 1 个 Issue
- Issue 内容包含：
  - 目标
  - 验收标准
  - 关联模块（如 Dashboard / Runs）
- 完成后关闭 Issue，并在 commit 或 PR 中引用 `#Issue编号`

> 本项目采用 Issue 作为过程管理工具，而非复杂的项目管理系统。

## 五、整体开发流程（设计先行）

### 5.1 设计阶段（必须完成后才能开发）

在编码前，需完成以下设计文档并冻结：

1. 页面与信息架构（IA_PAGES）
2. 数据库结构设计（DB_SCHEMA）
3. API 接口契约（API_SPEC）
4. AI 开发上下文约束（AI_CONTEXT）

设计阶段的目标是：**减少开发过程中的反复修改与返工**。

## 六、AI 辅助开发流程（里程碑制）

### 6.1 开发原则

- AI 仅作为实现工具，**不得突破已冻结设计**
- 每个阶段只完成一类明确目标
- 每个阶段结束必须可运行、可验证

### 6.2 里程碑划分（对应阶段性提交）

**里程碑 1：后端基础骨架**

- Spring Boot 项目初始化
- 数据库连接
- 基础实体与 Health 接口
   `commit: chore: bootstrap backend skeleton`

**里程碑 2：基础资源管理**

- Project / Tag / MetricDef CRUD
   `commit: feat: project tag metric_def crud`

**里程碑 3：实验记录管理**

- Run CRUD
- Run + Metrics + Tags 事务处理
   `commit: feat: run crud with metrics and tags`

**里程碑 4：Dashboard 聚合接口**

- summary / trend / distribution / top
   `commit: feat: dashboard aggregation apis`

**里程碑 5：前端基础结构**

- Vue 项目初始化
- Layout、Router、API 封装
   `commit: chore: bootstrap frontend layout`

**里程碑 6：核心页面实现**

- Runs 页面（列表 + 动态指标表单）
- Dashboard 页面（图表 + 筛选）
   `commit: feat: runs and dashboard pages`

**收尾阶段（强烈建议）**

- 性能与体验优化（loading、缓存、分页）
- 文档补全与截图
   `commit: perf: polish performance`
   `commit: docs: finalize documents`
