# IA_PAGES — 信息架构与页面设计

项目名称：ExperimentalAssistant  
系统定位：实验数据管理与可视化辅助系统（Web Dashboard 形态）

---

## 一、整体页面结构

系统采用典型后台管理布局：
- 左侧导航栏
- 顶部操作栏
- 主内容区域

页面数量控制在 5 个以内，保证功能完整且开发可控。

---

## 二、页面列表与功能说明

### 1. Dashboard（核心页面）

**功能定位**：  
对实验数据进行聚合分析与可视化展示，是系统的核心页面。

**页面模块：**
- 顶部筛选区：
  - Project
  - 日期范围
  - 实验状态（可选）
  - 指标选择（acc / loss / f1）
- 指标卡（KPI）：
  - 实验总数
  - 最近 7 天实验数
  - 成功率
  - 当前筛选条件下的最佳指标值
- 趋势图（Trend）：
  - 按天展示实验指标的每日最佳值（Best-of-day）
  - 根据指标方向（MAX / MIN）自动选择统计方式
- 分布图（Distribution）：
  - 按模型 / 数据集 / 标签统计实验数量分布
- TopN 表格：
  - 按指标排序展示最佳实验记录

---

### 2. Projects（项目管理）

**功能定位**：  
管理实验所属项目，用于对实验记录进行逻辑分组。

**页面模块：**
- 项目列表（分页）
- 新增 / 编辑 / 删除项目
- 展示项目下实验数量（辅助信息）

---

### 3. Runs（实验记录管理）

**功能定位**：  
管理实验记录（Run）及其对应的多项实验指标。

**页面模块：**
- 实验记录列表：
  - 分页
  - 关键词搜索
  - 按 Project / 状态 / 日期筛选
- 新增 / 编辑实验记录：
  - 基础信息（项目、模型、数据集、超参数等）
  - 动态指标表单（支持 acc / loss / f1 等多个指标）
  - 标签选择
- 查看实验详情：
  - 展示实验基础信息
  - 展示全部指标与标签

> 本系统采用**手动动态指标录入方式**，不支持 CSV 导入。

---

### 4. Management（管理页面）

**功能定位**：  
管理全局基础数据（元数据）。

**页面模块：**
- 标签管理 (Tags)
- 指标定义管理 (Metric Defs)
- 模板管理 (Templates)

---

### 5. AI Chat（独立 AI 页面）

**功能定位**：  
独立的 AI 辅助分析面板，支持通过虚拟文件树浏览 Run/Notes，并通过拖拽方式引用数据进行对话分析。**此页面为辅助功能，不影响主线业务流程。**

**页面模块：**
- 左侧虚拟文件树：
  - `run/`：浏览和搜索实验记录
  - `notes/`：管理项目笔记（本地存储）
- 中间对话区：
  - ChatGPT 风格聊天界面
  - 支持拖拽引用块 (Reference Chips)
- 后端代理：
  - 通过 `/api/ai/chat` 代理调用通义千问 (OpenAI-compatible)

---

### 6. About（说明页面）

**功能定位**：  
展示系统说明、技术选型与使用说明入口。

---

## 三、页面设计原则

- 页面数量精简，避免冗余
- 核心功能集中在 Dashboard 与 Runs
- 所有列表均支持分页与加载状态
- 可视化优先，管理功能为辅
